üìå The Three Notations at a Glance

These three notations describe how an algorithm's running time relates to a bounding function. Think of them as different ways to describe "speed limits" for algorithms.

üìù Worked Examples

Example 1: Linear Search

For an array of size n:

  • Best case: Element found at first position ‚Üí Œ©(1)
  • Worst case: Element at end or not present ‚Üí O(n)
  • We say: Linear search is O(n) because we describe worst-case guarantees

Note: We cannot say Œò(n) because best and worst cases differ!

Example 2: Merge Sort

For an array of size n:

  • Best case: Œò(n log n)
  • Worst case: Œò(n log n)
  • We say: Merge sort is Œò(n log n) because it always takes the same time regardless of input!

Since best = worst, we can use Theta notation.

üìå Summary

üéØ Key Takeaways
  • Big O (O) ‚Äî Upper bound, "at most", describes worst-case
  • Big Omega (Œ©) ‚Äî Lower bound, "at least", describes best-case
  • Big Theta (Œò) ‚Äî Tight bound, "exactly", when upper = lower
  • Use Œò when you know the exact growth rate; use O when you only have an upper bound guarantee
  • In practice, most people say "O(n)" even when they mean "Œò(n)" ‚Äî be precise when it matters!
